새 참여 코드를 발급할까요?

새 코드가 저장되는 즉시 이전 코드는 무효가 됩니다. 이미 참여한 학생의 멤버십에는 영향이 없습니다.

Course를 처음 만든 owner만 실행할 수 있고, 참여 코드는 만료되지 않으며 변경 이력은 저장하지 않습니다.

새 코드를 발급하지 못했습니다

기존 코드 ALGORT를 그대로 유지합니다.

실패한 요청은 이전 코드를 무효화하거나 새 코드를 표시하지 않습니다.

class 제목 수정

READY 상태에서도 날짜와 상태 시각은 바꾸지 않고 제목만 수정합니다.

비워 저장하면 서버 자동 제목을 사용합니다. 정확한 자동 제목 문자열은 아직 미정이므로 Prototype에는 형식 미정으로 표시합니다.

class를 삭제할까요?

선택한 class의 READY 기록을 삭제합니다.

이 동작은 `READY`와 `COMPLETED`에서만 제공합니다. `LIVE`와 `PROCESSING`에는 삭제 control 자체가 없습니다.

알고리즘 Course를 삭제할까요?

이 Dialog는 `current_session=null`인 안전한 목 fixture에서만 열립니다.

Course에는 종료·보관 상태나 owner 이전이 없습니다. active class가 있는 경우의 삭제 정책은 미정이라 이 성공 fixture를 제공하지 않습니다.
