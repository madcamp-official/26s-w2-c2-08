# GOAL: God Of All Lectures 기능 명세서

## 1. 문서 목적

본 문서는 GOAL 서비스가 사용자에게 제공해야 하는 기능과 동작 규칙을 정의한다. 구현 기술과 내부 구조는 [기술명세서](../architecture/기술명세서.md)에서 별도로 다룬다.

GOAL은 강의 중 발생하는 음성, 질문과 답변을 실시간으로 기록하고, 수업 종료 후 이를 AI 기반 복습 자료로 정리하는 강의 보조 서비스이다.

## 2. 계정과 Course 역할

| 구분 | 역할 |
|---|---|
| 인증 사용자 | 계정 전역 역할 없이 Course를 생성하거나 참여 코드로 Course에 참여한다. |
| Course 교수자 | Course를 생성한 유일한 owner로서 class와 참여 코드를 관리하고 학생 질문에 답변한다. Course당 한 명이며 이전할 수 없다. |
| Course 학생 | 참여 코드로 해당 Course에 참여해 강의 기록을 확인하고, 익명 질문과 AI 채팅을 이용한다. |

### 3.4 Course·class 관리

| ID | 기능 | 요구사항 |
|---|---|---|
| MGT-T-01 | 참여 코드 회전 | Course owner는 기존 코드를 즉시 무효화하고 새 `[A-Z]{6}` 코드를 발급할 수 있다. 회전 감사·이력은 보관하거나 제공하지 않는다. |
| MGT-T-02 | Course 삭제 | 유일한 Course owner는 Course를 삭제할 수 있다. Course 종료·보관과 owner 이전은 제공하지 않는다. |
| MGT-T-03 | class 제목 수정 | Course owner는 `READY`, `LIVE`, `PROCESSING`, `COMPLETED` class의 제목을 수정할 수 있다. 빈 제목은 서버 자동 제목으로 바뀐다. |
| MGT-T-04 | class 삭제 | Course owner는 `READY`와 `COMPLETED` class를 삭제할 수 있다. `LIVE`는 종료한 뒤 `PROCESSING` 완료를 기다려 삭제하고, `PROCESSING`에서는 바로 삭제하지 않는다. |
| MGT-T-05 | 강의자료 관리 | Course owner는 Session `READY`·`LIVE`·`COMPLETED`에서 PDF를 추가·삭제하고 상태를 확인할 수 있다. Session `PROCESSING`에서는 추가·삭제할 수 없다. |

| SYS-07 | AI 답변 범위 제한 | AI는 현재 세션에 연결되어 처리 완료된 `READY` PDF, Transcript와 Q&A를 우선 근거로 답변한다. |

## 5. 주요 동작 규칙

### 수업방과 세션

- 로그인 계정은 전역 역할을 갖지 않으며, 모든 인증 사용자는 Course를 생성하거나 참여 코드로 Course에 참여할 수 있다.
- Course당 교수자 owner는 생성자 한 명이며 추가·이전할 수 없다. Course owner만 class를 생성하고 시작·종료하며 Course와 class 관리 동작을 수행한다.
- 참여 코드는 입력의 앞뒤 공백을 제거하고 대문자로 정규한 `[A-Z]{6}`이다. 구분자를 허용하지 않고 자동 만료되지 않으며, owner만 회전할 수 있다.
- 참여 코드를 회전하면 기존 코드는 즉시 무효화하고 회전 감사·이력을 보관·표시하지 않는다.
- Course에는 `READY`, `LIVE`, `PROCESSING` 상태를 통틀어 active class가 최대 하나만 있다. Course의 `current_session`은 이 단일 active class 또는 `null`이다.
- active class가 `COMPLETED`가 된 뒤에만 다음 class를 생성할 수 있다. 동시 생성 경쟁은 하나만 성공해야 한다.
- 같은 `lecture_date`의 class를 여러 개 만들 수 있다. 완료 class 목록은 `started_at` 내림차순으로 정렬하고 날짜와 실제 시작 시각을 함께 표시한다.
- class 제목은 모든 상태에서 수정할 수 있고, trim 후 비어 있으면 Course 제목·class 날짜·시각을 포함한 서버 자동 제목을 지정한다. 정확한 문자열 형식, `READY`에서 사용할 시각 원장과 timezone은 미정이다.
- `lecture_date`, `started_at`, `ended_at`, `completed_at`은 사용자가 수정할 수 없다.
- Course는 종료·보관 상태를 갖지 않고 owner가 삭제한다. active class가 있는 Course 삭제와 삭제 후 복구의 세부 정책은 미정이다.
- `READY`와 `COMPLETED` class는 owner가 삭제할 수 있다. `LIVE`는 먼저 종료하고 `PROCESSING` 완료 후 삭제하며 `PROCESSING`은 완료까지 삭제하지 않는다.
- 종료된 세션에는 새 질문이나 반응을 등록할 수 없다.
- 세션의 사용자 표시 상태는 `시작 전`, `진행 중`, `정리 중`, `완료`로 구분한다.

### 강의자료

- 한 Session에는 삭제되지 않고 연결된 active PDF를 최대 10개까지 둘 수 있다. `UPLOADED`, `PROCESSING`, `READY`, `FAILED` 상태의 연결된 Material은 모두 이 제한에 포함한다.
- PDF 하나의 최대 크기는 decimal `100,000,000 bytes`다. 동일 content의 PDF를 다시 업로드할 수 있으며 content 중복을 오류로 처리하지 않는다.
- 같은 표시 이름이 있으면 확장자를 제외한 basename 뒤에 ` (1)`, ` (2)`를 업로드 시점에 안정적으로 붙인다. 예를 들어 `강의자료.pdf`, `강의자료 (1).pdf`, `강의자료 (2).pdf`로 표시하며, 중간 자료를 삭제해도 남은 이름을 재번호화하지 않는다.
- Course owner는 Session이 `READY`, `LIVE`, `COMPLETED`일 때 PDF를 업로드하거나 삭제할 수 있다. Session `PROCESSING`에서는 업로드와 삭제를 모두 거부한다.
- class 시작 시 현재 연결된 Material 중 `PROCESSING`이 하나라도 있으면 시작을 거부한다. 연결된 자료가 없거나 `READY`, `UPLOADED`, `FAILED`만 있으면 시작할 수 있다.
- AI 참고 자료는 현재 연결된 `READY` Material로 한정한다. `UPLOADED`, `PROCESSING`, `FAILED`와 삭제된 Material의 Knowledge 자료는 새 AI 검색에 사용하지 않는다.
- `FAILED` Material은 사용 가능한 업로드 자료 수나 AI 참고 자료로 표현하지 않고, 업로드에 실패해 사용할 수 없음을 명확히 표시한다. 재시도 가능 여부와 삭제 제공 여부는 Job과 Session 상태 규칙을 따른다.
- 삭제 transaction이 확정되면 Material은 목록·열람과 새 AI 검색에서 즉시 분리된다. object와 Knowledge 자료의 물리 정리는 백그라운드에서 진행하며 정리 실패가 삭제를 되돌리거나 자료를 다시 노출하지 않는다.
- 중복 업로드·삭제 요청의 재실행은 공통 멱등성 규칙을 따른다. 상태 또는 version 경합으로 삭제를 확정하지 못하면 최신 목록을 조회한 뒤 다시 시도한다.
- 삭제 전에 생성된 AI Evidence를 삭제 후 보관하거나 표시하는 세부 정책은 미정이다. 다만 삭제된 Material 원문으로 이동하는 동작은 제공하지 않는다.

### 익명 질문과 반응

- 학생의 이름과 계정 정보는 교수자와 다른 학생에게 표시하지 않는다.
- 서비스는 중복 반응 방지 등을 위해 질문 작성자와 반응 사용자를 내부적으로 식별할 수 있다.
- 한 학생은 같은 질문에 ‘나도 궁금해요’를 한 번만 추가할 수 있으며 취소할 수 있다.
- 질문 클러스터링 이후에도 원본 질문을 각각 확인할 수 있어야 한다.
- 클러스터링에 실패해도 질문 작성, 조회와 답변 기능은 계속 사용할 수 있어야 한다.

### Transcript와 답변

- 아직 확정되지 않은 실시간 Transcript와 저장된 확정 Transcript를 화면에서 구분한다.
- 교수자가 답변 대상을 선택한 시점부터 완료한 시점까지의 확정 Transcript를 답변 후보로 연결한다.
- 교수자는 자동 연결 결과를 확인한 후 답변 완료 상태로 변경할 수 있다.

### AI 기능

- 실시간 요약은 요청 시점까지의 Transcript와 해당 세션에 현재 연결된 `READY` PDF를 기준으로 생성한다.
- 복습 채팅은 선택한 세션에 현재 연결된 `READY` 자료와 기록만 기본 답변 범위로 사용한다.
- 강의 기록에서 근거를 찾지 못하면 확인할 수 없는 내용임을 사용자에게 알린다.
- AI 작업 중에는 진행 상태를 표시하고 실패한 작업은 재시도할 수 있게 한다.

### 수업 종료

- 강의 종료 후 기록이 생성되는 동안 사용자에게 ‘정리 중’ 상태를 표시한다.
- 일부 AI 작업이 실패하더라도 현재 연결된 열람 가능한 PDF, Transcript와 Q&A는 먼저 열람할 수 있어야 한다.
- 후처리가 완료되면 강의 기록 페이지에서 요약, 질문·답변과 복습 채팅을 제공한다.

## 6. MVP 완료 조건

1. 인증 사용자가 Course를 생성해 유일한 교수자 owner가 되고, 다른 인증 사용자가 참여 코드로 해당 Course의 학생이 될 수 있다.
2. Course owner가 active class가 없을 때 class를 만들고 PDF를 업로드할 수 있다.
3. Transcript, 질문, 반응과 답변 상태가 새로고침 없이 참여자 화면에 반영된다.
4. 학생이 익명 질문을 작성하고 다른 질문에 한 번만 반응할 수 있다.
5. 유사 질문이 클러스터로 표시되고 원본 질문도 확인할 수 있다.
6. 교수자가 질문을 선택하고 음성 답변 Transcript를 연결할 수 있다.
7. AI가 현재까지의 강의를 요약하고 현재 연결된 `READY` PDF·Transcript 기반 질문에 답변한다.
8. 강의 종료 후 Transcript, 요약과 질문·답변이 포함된 기록 페이지가 생성된다.
9. 학생이 해당 강의 기록을 근거로 AI와 복습 대화를 할 수 있다.
10. 권한이 없는 사용자는 수업 자료와 기록에 접근할 수 없다.
11. 실패한 AI 작업의 상태를 확인하고 다시 실행할 수 있다.
12. Course owner가 참여 코드를 회전하면 기존 코드는 즉시 무효화되고 새 `[A-Z]{6}` 코드만 참여에 사용할 수 있다.
13. Course에는 active class가 최대 하나만 존재하고 `current_session`은 해당 class 또는 `null`이며, 완료 후에만 다음 class를 생성할 수 있다.
14. 같은 `lecture_date`의 완료 class를 실제 시작 시각과 함께 `started_at` 내림차순으로 구분해 확인할 수 있다.
15. Course owner가 모든 상태의 class 제목을 수정할 수 있고, 빈 제목은 자동 제목으로 바뀌며 날짜와 상태 시각은 수정할 수 없다.
16. Course owner가 `READY`·`COMPLETED` class를 삭제할 수 있고, `LIVE`·`PROCESSING`에서는 허용된 종료·완료 절차를 거친 뒤 삭제할 수 있다.
17. Course owner가 Session `READY`·`LIVE`·`COMPLETED`에서 최대 10개의 PDF를 추가·삭제할 수 있고, 각 파일은 decimal `100,000,000 bytes`를 넘지 않는다.
18. PDF가 없거나 `READY`·`UPLOADED`·`FAILED`만 있으면 class를 시작할 수 있고, 연결된 `PROCESSING` Material이 하나라도 있으면 시작할 수 없다.
19. Material 삭제가 확정되면 목록·열람·새 AI 검색에서 즉시 사라지고 백그라운드 정리 실패에도 다시 노출되지 않는다.
20. 동일 content 재업로드와 안정적인 표시 이름 suffix를 지원하며, `FAILED` 자료는 사용할 수 없는 업로드 실패로 명확히 표시한다.
